Grid Composition A P5.JS study inspired by Sol LeWitt

Inspired by Sol LeWitt, Wall Drawing series
p5.js study by Your Name

Step 1 — Canvas Setup

In this sketch I began by creating a 600 x 600 canvas, setting the background to white and adjusting stroke weight (sw) so it would more closely resemble the reference. I defined the value of ‘inset’, and drew a border around the parameter of the canvas using rect() where the x and y coordinates are set to inset. The reason for this is so that the full width of the border is contained within the canvas.

Step 2 — Dividing the canvas into quadrants

First I defined the value of half (let half = width / 2 e.g. 300). I then used this value as well as the full value of width and height to draw two lines, one that divides the canvas vertically, and one that divides the canvas horizontally. The result is that the canvas is now divided into four equal quadrants.

I defined the value of spacing e.g. the space between each line that will be drawn, and using the value of half, created a while loop that fills the quadrant with vertical parallel lines with 20px of spacing in between them. Step 4 — Centering the Vertical Lines

To resolve the issue of unevenly spaced lines I calculated a boundary by first defining a value I called ‘clearance’ (let clearance = spacing / 2 ), which is the value alloted the left and right side. Using this value I then defined a series of parametres that make up the quadrant, accounting for the left and right side of the quadrant to determine the usable width. Additionally, I created the offsetL which calculated how much the lines would need to be offset to compensate for the black border. This resulted in the lines having even spacing on either side of them.

Step 6 — Drawing diagonal lines in the bottom, left quadrant

I implemented a while loop to draw diagonal lines across the bottom left quadrant. The diagonal line is achieved by the two coordinate points for each line, which increase while maintaining the same difference to create parallel diagonal lines.

This however created multiple changes where initially the lines exceeded their designated area and the spacing between lines appeared smaller.

Step 7 — Implementing Clipping Contraints

To solve the issue of the lines exceeding their area I implemented clipping constraints. This first required me to define the area of the quadrant in a similar way to that which I’d done previously. I used a push() and pop(), enabling me to introduce new rules inside. I then added clipping constraints which are defined within the new variable clip(), so that the lines stay within the quadrant.
